INSTALL_OBJECT_NAME¶

在 4.2 版被加入.

設定來源檔案的已安裝目標名稱（不含副檔名）。空字串值會停用自訂目標命名。該值必須是相對路徑，且不得包含特殊目錄組件（例如 ..）。

請注意，在某些 INSTALL_OBJECT_NAME_STRATEGY 策略中，目標名稱可能不會按原樣使用。它可能會根據策略的需要進行更改以實現其目標。

此屬性支援 生成器表達式。

備註

CMake 不會對目標內的名稱衝突進行防護。使用此屬性時，必須在專案程式碼中避免衝突。CMake 會產生一些也會建立目標檔的來源檔案，這些檔案可能與給定的自訂名稱衝突。其中包括：

  • 產生的 PCH 來源檔案 (cmake_pch)

  • 產生的 Unity 編譯檔案 (unity_...)

  • Qt autogen 來源 (moc_compilations.cpp)
